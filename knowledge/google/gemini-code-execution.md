## Overview

The Gemini API's code execution feature enables models to "generate and run Python code based on plain-text instructions" and output visualizations. The model iteratively refines results based on execution feedback.

## Setup


```python
from google import genai
from google.genai import types

client = genai.Client(api_key='YOUR_API_KEY')
MODEL_ID = "gemini-2.5-flash"
```

## Basic Code Execution

```python
prompt = """What is the sum of the first 50 prime numbers?
Generate and run code for the calculation, then explain the result."""

response = client.models.generate_content(
    model=MODEL_ID,
    contents=prompt,
    config=types.GenerateContentConfig(
        tools=[types.Tool(
            code_execution=types.ToolCodeExecution
        )]
    )
)

print(response.text)
```

## Response Structure

Model responses with code execution contain multiple part types:

| Part Type | Description |
|-----------|-------------|
| `text` | Inline text generated by model |
| `executable_code` | Python code meant for execution |
| `code_execution_result` | Output from executed code |
| `inline_data` | Media/images generated by the code |

```python
response = client.models.generate_content(
    model=MODEL_ID,
    contents="Plot a sine wave",
    config=types.GenerateContentConfig(
        tools=[types.Tool(code_execution=types.ToolCodeExecution)]
    )
)

# Inspect response parts
for part in response.candidates[0].content.parts:
    if part.text:
        print(f"TEXT: {part.text}")
    if part.executable_code:
        print(f"CODE: {part.executable_code.code}")
    if part.code_execution_result:
        print(f"OUTPUT: {part.code_execution_result.output}")
    if part.inline_data:
        print(f"MEDIA: {part.inline_data.mime_type}")
```

## Multi-Turn Code Execution (Chat)

```python
chat = client.chats.create(
    model=MODEL_ID,
    config=types.GenerateContentConfig(
        tools=[types.Tool(code_execution=types.ToolCodeExecution)]
    )
)

# First turn
response = chat.send_message("Generate a dataset of 100 random numbers with a normal distribution")
print(response.text)

# Follow-up using the generated data context
response = chat.send_message("Now fit a normal distribution to this data and display the fit")
print(response.text)

# Another follow-up
response = chat.send_message("What is the mean and standard deviation?")
print(response.text)
```

## Iterative Refinement

The model automatically refines code when execution fails:

```python
response = client.models.generate_content(
    model=MODEL_ID,
    contents="Write Python code to solve the traveling salesman problem for 10 cities",
    config=types.GenerateContentConfig(
        tools=[types.Tool(code_execution=types.ToolCodeExecution)]
    )
)
# Model will generate code, execute it, fix any errors, and return working result
```

## Via Interactions API

```python
interaction = client.interactions.create(
    model='gemini-2.5-flash',
    input='Calculate the 50th Fibonacci number.',
    tools=[{'type': 'code_execution'}]
)
print(interaction.outputs[-1].text)
```

## Key Features

- **Iterative learning**: Model refines results based on execution feedback
- **Graph generation**: Create visualizations via matplotlib, plotly, etc.
- **Data analysis**: Process local files or uploaded datasets
- **Chat integration**: Use code execution across multi-turn conversations
- **Multimodal support**: Works with text and image inputs

## Supported Models

- `gemini-2.5-flash` — Recommended
- `gemini-2.5-pro` — Most capable
- `gemini-2.5-flash-lite` — Lightweight
- Note: Thinking models (2.5 series) require additional time

## Available Python Libraries in Execution Environment

The code execution environment supports standard Python libraries including:
- `numpy`
- `pandas`
- `matplotlib`
- `scipy`
- Standard library modules

## Important Notes

- Code execution runs in a sandboxed environment
- Only Python is supported (not other languages)
- Generated files/plots are returned as `inline_data` parts
- The model can iterate automatically on failed executions
